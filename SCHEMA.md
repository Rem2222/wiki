# SCHEMA.md — Правила вики

Это конфигурационный файл для LLM. Описывает как поддерживать вики.

## Структура

```
wiki/
├── raw/            — исходники (статьи, документы, скриншоты). LLM читает, не меняет.
├── assets/         — картинки, вложения
├── wiki/
│   ├── people/     — люди (кто есть кто)
│   ├── tech/       — технологии, инструменты, языки
│   ├── projects/   — проекты (личные, рабочие)
│   ├── concepts/   — концепции, идеи, методологии
│   ├── books/      — книги, курсы, подкасты
│   └── misc/       — всё что не влезло в остальное
├── index.md        — каталог всего
├── log.md          — журнал действий
└── SCHEMA.md       — этот файл
```

## Формат страниц

Каждая страница — markdown-файл с YAML frontmatter:

```markdown
---
title: Название
type: person | tech | project | concept | book | misc
created: 2026-04-05
updated: 2026-04-05
tags: [тег1, тег2]
sources: [raw/название.md]
related: [[Связанная страница1]], [[Связанная страница2]]
---

# Название

Краткое описание (2-3 предложения).

## Детали
...

## Связи
...
```

## Правила при импорте (ingest)

1. Прочитать исходник
2. Обсудить ключевые выводы с Романом
3. Создать/обновить страницу в wiki/
4. Расставить перекрёстные ссылки `[[ ]]` на связанные страницы
5. Обновить index.md (добавить/обновить запись)
6. Добавить запись в log.md
7. Обновить related на затронутых страницах

## Правила при запросе (query)

1. Сначала прочитать index.md чтобы найти релевантные страницы
2. Прочитать найденные страницы
3. Синтезировать ответ с цитатами
4. Если ответ ценный — сохранить как новую страницу в wiki/

## Правила при проверке (lint)

Периодически (по запросу Романа или когда wiki разрастётся):
- Искать противоречия между страницами
- Искать устаревшие утверждения
- Искать страницы-сироты без входящих ссылок
- Искать упоминания без собственных страниц
- Искать пропущенные перекрёстные ссылки

## index.md

Каталог всего в wiki. Формат:

```markdown
## Люди
- [[имя]] — краткое описание (дата добавления)

## Технологии
- [[название]] — краткое описание (дата добавления)
```

LLM обновляет при каждом импорте.

## log.md

Журнал действий (append-only). Формат:

```markdown
## [2026-04-05] ingest | Название источника
Краткое описание что сделано. Какие страницы созданы/обновлены.

## [2026-04-05] query | Вопрос
Кратко о вопросе и ответе. Сохранено ли в wiki.
```

## Важно

- **Не галлюцинировать** — если не уверен, не пиши. Спроси Романа.
- **Спрашивать перед созданием** — если не ясно куда отнести, спросить.
- **Ссылки > дублирование** — лучше `[[страница]]` чем копировать текст.
- **Обновлять updated** — при любом изменении страницы.
