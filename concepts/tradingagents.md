# TradingAgents

Мультиагентный LLM-фреймворк для алгоритмической торговли на фондовом рынке. Симулирует устройство реальной торговой фирмы с распределением ролей и механизмом дебатов.

## Суть

TradingAgents (arXiv:2412.20138, UCLA + MIT) — первый фреймворк, где LLM-агенты играют роли аналитиков, исследователей, трейдеров и риск-менеджеров реальной торговой фирмы. Вместо одного LLM-трейдера — команда специалистов, которая дебатирует и приходит к консенсусу.

## Архитектура (5 команд)

```
┌─────────────────────────────────────────────────────────┐
│                    TradingAgents                         │
│                                                         │
│  ┌─────────────┐     ┌─────────────┐    ┌───────────┐ │
│  │Analyst Team │────▶│Research Team│───▶│  Trader   │ │
│  │(4 агента)   │     │(Bull+Bear)  │    │           │ │
│  └─────────────┘     └─────────────┘    └─────┬─────┘ │
│                                                │       │
│  ┌─────────────────┐                 ┌────────▼────┐  │
│  │Risk Management  │◀────────────────│Fund Manager │  │
│  │                 │                 │             │  │
│  └─────────────────┘                 └─────────────┘  │
└─────────────────────────────────────────────────────────┘
```

### Analyst Team (4 агента)
- **Fundamental Analyst** — фундаментальный анализ (P/E, EPS, баланс)
- **Sentiment Analyst** — настроения из соцсетей
- **News Analyst** — новости и макроэкономика
- **Technical Analyst** — индикаторы (MACD, KDJ, RSI, SMA)

### Research Team (Bull vs Bear)
Два исследователя с противоположными позициями. Дебаты обеспечивают сбалансированный взгляд — ни бычий, ни медвежий уклон не доминирует.

### Trader
Принимает финальное торговое решение (buy/sell/hold), размер позиции, тайминг.

### Risk Management
Контролирует волатильность, ликвидность, максимальную просадку. Ограничивает риск-аппетит команды.

### Fund Manager
Утверждает и исполняет сделку.

## Ключевые механизмы

### ReAct Prompting
Все агенты используют ReAct (Reasoning + Acting) — цикл: подумать → извлеки данные → действуй → проанализируй результат. Это даёт прозрачность почему агент принял то или иное решение.

### Structured Communication
В отличие от предыдущих фреймворков с неструктурированным диалогом, здесь:
- Аналитики → структурированные отчёты
- Трейдеры → сигналы с обоснованием (buy/sell/hold + rationale)
- Дебаты (Research, Risk) → естественный язык для глубокого reasoning

### Quick vs Deep Thinking
- Быстрые модели (GPT-4o-mini) → data retrieval, простая аналитика
- Глубокие модели (GPT-4o, Claude) → дебаты, финальные решения

## Результаты бэктеста

**Период:** июнь–ноябрь 2024, данные: янв–март 2024
**Акции:** AAPL, GOOGL, AMZN

| Метрика | TradingAgents | Best Baseline | Улучшение |
|---------|-------------|---------------|-----------|
| AAPL Cumulative Return | **+26.6%** | -5.2% (B&H) | +31.8pp |
| GOOGL Cumulative Return | **+24.4%** | 7.8% (B&H) | +16.6pp |
| AMZN Cumulative Return | **+23.2%** | 17.1% (B&H) | +6.1pp |
| AAPL Sharpe Ratio | **8.21** | 1.35 (B&H) | ×6 |
| AAPL Max Drawdown | **0.91%** | 11.9% (B&H) | −13× |

Превосходит все rule-based стратегии (MACD, KDJ+RSI, ZMR, SMA) по всем метрикам.

## Данные

- Исторические цены акций
- Новости и макроэкономические индикаторы
- Настроения из соцсетей (Twitter/X)
- Внушительные сделки (insider transactions)
- Финансовые отчёты (10-K, 10-Q)
- Технические индикаторы

## Чем отличается от других

| | TradingAgents | Обычные LLM-трейдеры |
|--|--|--|
| Архитектура | Команда специалистов | Один LLM |
| Обоснование | Debates + структурные отчёты | Непрозрачные |
| Риск-менеджмент | Отдельная команда | Отсутствует |
| Данные | Мульти-модальные (цены, новости, соцсети) | Обычно только цены |
| Работает на | Любых LLM через API | Нужны GPU |

## Потенциал для Романа

- **Архитектурный паттерн** — можно адаптировать для любой задачи, где нужна команда экспертов с дебатами
- **ReAct + structured output** — хороший шаблон для мультиагентных систем в 1С или других доменах
- **Multi-agent trading** — если будет интерес к алгоритмической торговле, это готовая база
