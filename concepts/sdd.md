# Spec-Driven Development (SDD)

Методология разработки ПО, при которой **спецификация** создаётся до кода и является единственным источником истины. Код рассматривается как артефакт, сгенерированный из спецификации.

## Суть

1. Сначала пишется подробная спецификация (что система должна делать)
2. LLM-агент (Claude Code, Hermes Agent и т.д.) реализует код строго по спецификации
3. Любые изменения начинаются с изменения спецификации

## SDD-инструменты

- [[tech/openspec]] — человекочитаемые документы, автоматическая корректировка спецификации
- [[tech/gsd]] — интервью-ориентированный подход, лучшая декомпозиция
- [[tech/bmad-method]] — архитектура прежде всего, исследовательские сценарии
- [[tech/spec-kit]] — starter kit от GitHub
