# Оркестратор v3 — Руководство

## Что это

Оркестратор — ИИ-ассистент, который превращает идеи в задачи, распределяет работу между субагентами и отслеживает результаты.

**v3 отличие:** Использует **Multica** вместо Linear для task tracking. Multica — локальная система на твоём VPS.

---

## Быстрый старт

### Активация

Скажи: **"запусти Оркестратора v3"**

Ассистент прочитает инструкции и станет Оркестратором.

### Как работает

```
Роман → "сделай API для авторизации"
         ↓
    Оркестратор
         ↓
    1. Создаёт задачу в Multica (MUL-N)
    2. Пишет SPEC.md с требованиями
    3. Спавнит субагента для задачи
         ↓
    Субагент делает работу
         ↓
    Оркестратор проверяет результат
         ↓
    Уведомляет тебя
```

## Multica — CLI команды

### Проверка

```bash
multica auth status          # авторизация OK?
multica daemon status        # демон запущен?
multica --version           # версия
```

### Задачи

```bash
## Список задач
multica issue list

## Создать задачу
multica issue create --title "Название" --description "Описание" --status todo

## Изменить статус
multica issue update MUL-17 --status done

## Назначить на агента
multica issue assign MUL-17 --agent bf306a1f-2f94-4d4d-9b9a-cdcb7bf5f1a0
```

**Статусы:** backlog, todo, in_progress, done

### Workflow

```bash
## 1. Создать задачу
multica issue create --title "Новый фича" --description "..." --status todo

## 2. Обновить когда работает
multica issue update MUL-18 --status in_progress

## 3. Завершить
multica issue update MUL-18 --status done
```

## Если сломалось

**Multica не отвечает:**
```bash
multica daemon stop && multica daemon start
```

**Browser UI 502:**
```bash
ssh root@81.17.100.103 "docker-compose -f /opt/multica/docker-compose.yml restart frontend"
```

**Tailscale не подключается:**
Открыть Tailscale app на Windows и подключиться.
