# OpenSpec

**GitHub:** https://github.com/nickscamara/openspec

Spec-Driven Development фреймворк. Философия: **«Specs that people actually read»**.

## Фишка

Самые человекочитаемые документы среди SDD-фреймворков. Автокорректировка спецификации при пересмотрах — документы сами обновляются по мере изменений.

## Особенности

- **Человекочитаемые документы** — спецификации понятны не только AI, но и людям
- **Автокорректировка** — при пересмотрах спецификация обновляется автоматически
- **Хорош для evolving проектов** — где требования меняются в процессе

## Минусы

- **Слабая декомпозиция** — разбиение задачи на этапы надо делать самостоятельно
- **Слабая исследовательская фаза** — надо явно сообщать, какие исследования провести
- **Меньше автоматизации** — больше ручной работы по структурированию

## Когда выбирать

✅ **Для squads с human-участниками** — люди хорошо читают OpenSpec-документы
✅ **Комбинация с GSD** — OpenSpec для обсуждения с людьми, GSD для декомпозиции агентам
❌ Для чисто агентных squads — GSD даёт лучшую декомпозицию

## Сравнение с другими SDD

| Аспект | OpenSpec | GSD | BMAD |
|--------|----------|-----|------|
| Декомпозиция | ❌ ручная | ✅ лучшая | 🟡 средняя |
| Исследование | ❌ слабое | 🟡 хорошее | ✅ лучшее |
| Читаемость | ✅ отличная | 🟡 средняя | 🟡 средняя |
| Автокорректировка | ✅ есть | ❌ нет | ❌ нет |
