## 1. Hermes Agent {#hermes-agent}

**GitHub:** [nousresearch/hermes-agent](https://github.com/nousresearch/hermes-agent) (~100K ⭐)

### Суть
Личный AI-ассистент с долгой памятью. Решает главную боль: каждый новый диалог — как с чистого листа. Hermes запоминает предыдущие диалоги через SQLite + FTS5 полнотекстовый поиск.

### Ключевые фичи
- **Долгая память:** MEMORY.md (факты о проекте, 2200 символов) + USER.md (предпочтения, 1375 символов). Жёсткий лимит заставляет сжимать информацию
- **Самообучение:** после задач с 5+ вызовами инструментов автоматически создаёт «скиллы» — переиспользуемые инструкции
- **Саморефлексия:** периодически сам себя спрашивает «не пора ли сохранить в долгую память?»
- **Провайдер-агностик:** работает с любой моделью через OpenAI-совместимый API (включая Ollama локально)
- **MCP-интеграция** из коробки (Model Context Protocol)
- **15+ платформ** из одного процесса: Telegram, Discord, Slack, WhatsApp, Signal, DingTalk, SMS, Mattermost, Matrix, Email, Feishu/Lark, WeCom, WeChat и др.

### Плюсы
- Настоящий FTS5-поиск по всей истории диалогов
- Автоматическое создание скиллов
- Работает с локальными моделями
- Нативная MCP-поддержка

### Минусы
- Ресурсоёмкий (не «pip install», а микросервис)
- Сыроватость в管理系统 скиллов (дубли, странные названия)
- Требует времени на «воспитание» — не становится полезным за один диалог

### Вердикт
> «Отличная заготовка для личного Джарвиса, но будьте готовы, что первое время он будет тупить, как стажёр в понедельник — правда, в отличие от стажёра, он действительно учится на ошибках.»

## 2. DeerFlow {#deerflow}

**GitHub:** [bytedance/deer-flow](https://github.com/bytedance/deer-flow)
**Автор:** ByteDance (TikTok)

### Суть
Мультиагентный фреймворк с иерархической архитектурой «Супервайзер + суб-агенты». Супервайзер получает задачу, декомпозирует её, порождает суб-агентов, распределяет подзадачи, собирает результат.

DeerFlow 2.0 — полный перезапуск: первая версия была узким инструментом для «глубокого ресёрча», вторая — рантайм для мультиагентных систем.

### Ключевые фичи
- **Архитектура SuperAgent:** координатор → исследователь → кодер → background investigator. Контексты суб-агентов строго изолированы
- **Песочницы на Docker:** каждый суб-агент работает в изолированном Docker-контейнере с файловой системой, bash-терминалом и возможностью ставить пакеты
- **LangGraph под капотом:** можно залезть внутрь и допилить под себя
- **Скиллы из коробки:** исследование, генерация отчётов, слайды, веб-страницы, изображения, видео
- **Управление контекстом:** завершённые подзадачи автоматически суммируются

### Плюсы
- Чёткая иерархия вместо «плоского чата, где все орут одновременно»
- Реальные Docker-песочницы (не симуляция)
- Контекстное окно не раздувается

### Минусы
- Сложная настройка (API + веб-интерфейс + конфиги — не «npm start»)
- Избыточность для простых задач (как стрелять из пушки по воробьям)
- Нужен мощный GPU и топовые модели (ByteDance рекомендует 100k+ контекст)
- Супервайзер должен быть действительно умным — слабая модель спланирует ерунду

### Вердикт
> «DeerFlow — это ваш AI-проджект-менеджер, который действительно умеет декомпозировать задачи, но требует к себе уважения, хорошего GPU и пары вечеров на настройку.»

## 3. Multica {#multica}

**GitHub:** [multica-ai/multica](https://github.com/multica-ai/multica) (~10.7K ⭐ за 3 месяца)
**Старт:** январь 2026

### Суть
Jira/Linear/Trello, но заточенная под работу с AI-агентами. Превращает разрозненные запуски агентов в структурированный процесс: задачи, статусы, история, бюджет.

### Почему выстрелила
Идея «агент как тиммейт» оказалась очень резонансной. Поддерживает интеграцию с Claude Code, Codex, OpenClaw, OpenCode, Hermes, Gemini, Pi, Cursor Agent.

### Ключевые фичи
- **Наблюдаемость:** кто чем занят, какие задачи в работе, какие заблокированы
- **Управление бюджетом:** лимиты на токены
- **Скиллы накапливаются:** каждое успешное решение → переиспользуемый скилл
- **Селф-хостинг:** Docker Compose
- **Агенты пишут в комментарии:** если не хватает прав или требования непонятны — агент пишет комментарий и запрашивает помощь, не зависает в цикле

### Плюсы
- Прозрачность и контроль
- Поддержка многих AI-платформ

### Минусы
- Инфраструктурный проект (сам не решает задачи — только управляет)
- Очень молодой (0.1.x), активно развивается но до enterprise далеко
- Сложная архитектура: Next.js 16 + Go + PostgreSQL + pgvector

### Вердикт
> «Multica — это красивый и функциональный органайзер для вашего AI-зверинца, но самих зверей вам придётся искать, кормить и дрессировать отдельно.»

## 4. Claude Code Game Studios {#claude-code-game-studios}

**Надстройка над Claude Code (Anthropic)**

### Суть
Набор конфигураций для Claude Code, превращающий одну AI-сессию в полноценную виртуальную игровую студию. **48 специализированных агентов**, 37 воркфлоу-скиллов, 8 хуков, 11 правил.

Три уровня иерархии:
- **Директора:** креативный директор, техдир, продюсер
- **Лиды направлений:** дизайн, программирование, арт, звук, нарратив, QA
- **Специалисты:** геймплей-программист, специалист по доступности и др.

### Плюсы
- 48 агентов — продуманная структура, копирующая настоящую геймдев-студию
- 70+ слеш-команд: `/brainstorm`, `/design-system`, `/dev-story`, `/qa-plan`, `/team-combat` и др.
- Готовые пресеты под Godot 4, Unity (DOTS), Unreal Engine 5 (GAS, Blueprints)
- Хуки и правила при коммитах и завершении сессии

### Минусы
- Жёсткая завязка на Claude Code (без него бесполезен)
- Стоимость: API Anthropic стоит денег
- Сложность адаптации под другие домены (очень специфичный)
- Качество варьируется: прототип и играбельный билд — да, релиз в Steam — руками

### Вердикт
> «Показательная сборка Lego Technic на витрине: смотреть невероятно приятно, работает как часы, но повторить у себя — отдельный проект.»

## 5. MarkItDown {#markitdown}

**GitHub:** [microsoft/markitdown](https://github.com/microsoft/markitdown)
**Автор:** Microsoft

### Суть
Конвертер файлов различных форматов в Markdown. Простой, утилитарный инструмент.

Поддерживает: PowerPoint, PDF, Excel, Word и др.

### Особенности
- Open Source
- Microsoft-backed
- Конвертация без потери структуры

1. **Для личного ассистента** → Hermes Agent (долгая память, скиллы)
2. **Для сложных мультиагентных пайплайнов** → DeerFlow (Docker-песочницы, LangGraph)
