# LLM Wiki

Паттерн создания персональных баз знаний с помощью LLM. Вместо RAG (где LLM каждый раз ищет куски документов заново), LLM постепенно строит **постоянную вики** — структурированную, взаимосвязанную коллекцию markdown-файлов.

## Суть

Знания компилируются один раз и поддерживаются в актуальном состоянии, а не выводятся заново при каждом запросе. Перекрёстные ссылки уже расставлены. Противоречия уже отмечены. С каждым новым источником вики становится богаче.

**Ключевая метафора:** Obsidian — это IDE; LLM — программист; вики — кодовая база.

## Три уровня

| Уровень | Описание | Кто управляет |
|---------|----------|---------------|
| **Raw sources** | Исходные документы (статьи, научные работы, данные). Неизменны. | Человек |
| **Wiki** | Сгенерированные markdown-страницы (сводки, сущности, концепции, сравнения). | LLM |
| **Schema** | Правила и рабочие процессы (CLAUDE.md, AGENTS.md, SCHEMA.md). | Человек + LLM |

## Три процесса

### Ingest (Импорт)
1. Человек добавляет источник в raw/
2. LLM читает, обсуждает ключевые выводы
3. LLM создаёт/обновляет страницы в wiki
4. Обновляет index.md и log.md
5. Один источник может затронуть 10-15 страниц

### Query (Запрос)
1. Человек задаёт вопрос
2. LLM читает index.md → находит релевантные страницы
3. Синтезирует ответ с цитатами
4. **Хорошие ответы сохраняются обратно в wiki** как новые страницы

### Lint (Проверка)
Периодическая проверка целостности:
- Противоречия между страницами
- Устаревшие утверждения
- Страницы-сироты без входящих ссылок
- Упоминания без собственных страниц
- Пропущенные перекрёстные ссылки
- Пробелы в данных

## Ключевые файлы

- **index.md** — каталог всего (замещает RAG на ~100 источников)
- **log.md** — журнал действий (append-only, greppable)
- **SCHEMA.md** — правила для LLM

## Применение

- **Личное:** цели, здоровье, саморазвитие, дневники
- **Исследования:** научные работы, статьи, отчёты
- **Чтение книг:** персонажи, темы, сюжетные линии
- **Бизнес:** внутренняя вики, Slack-треды, встречи
- **Анализ:** конкуренты, due diligence, планирование

## Инструменты

- [[Obsidian]] — IDE для вики
- [[qmd]] — локальный поиск по markdown
- [[Obsidian Web Clipper]] — конвертация веб-статей в markdown
- [[Marp]] — презентации из markdown
- [[Dataview]] — запросы к метаданным страниц
- Git — версионность (wiki = git-репозиторий .md файлов)

## Историческая параллель

Перекликается с **[[Memex]]** Вэнивара Буша (1945) — личное курируемое хранилище знаний с ассоциативными связями. Буш не мог решить кто будет поддерживать систему. Теперь эту задачу берёт на себя LLM.
