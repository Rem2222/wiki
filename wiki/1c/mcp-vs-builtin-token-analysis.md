# MCP vs Built-in Tools: анализ потребления токенов

**Автор:** Антон Миняков  
**Проект:** Lombardis (3200+ BSL-файлов)  
**Дата:** 2026-04-06  
**Источник:** Бенчмарк задачи #1285

---

## Суть

MCP (Model Context Protocol) экономит **75-85% токенов** на семантических задачах в 1С по сравнению с встроенными инструментами (Grep, Read, Glob).

---

## Главный вывод

| Метрика | Built-in | MCP | Разница |
|---|---|---|---|
| Семантическая задача | ~750K токенов | ~175K токенов | **4.3x** |
| Время | ~60 сек | ~30 сек | **2x** |
| Точность | false positives | семантика | **MCP лучше** |

**С кэшированием:** MCP дешевле в ~2.5x по деньгам.

---

## Когда что использовать

| Задача | Инструмент | Причина |
|---|---|---|
| Понять структуру модуля | **MCP** | 1 вызов vs 5-10 |
| Найти кто вызывает метод | **MCP** | Без false positives |
| Узнать реквизиты объекта | **MCP** | Без парсинга XML |
| Валидация запроса 1С | **MCP** | Нет альтернативы |
| IDE autocomplete | **MCP** | Нет альтернативы |
| Найти строку по тексту | **Grep** | Быстрее |
| Прочитать конкретный файл | **Read** | Без разницы |

---

## Правило принятия решения

```
Вопрос семантический (структура, типы, вызовы, валидация)?
 → Да → MCP
 → Нет → Вопрос текстовый (найти строку, прочитать файл)?
 → Да → Grep/Read/Glob
 → Нет → MCP (по умолчанию)
```

## Ключевые метрики

### Количество вызовов — главный фактор

Каждый вызов инструмента — полный раундтрип. Контекст растёт квадратично:

```
Total_input ≈ n × C₀ + n(n-1)/2 × (R + O)
```

где `n` — количество вызовов.

### Сравнение для семантической задачи

| Тип задачи | Built-in | MCP | Экономия |
|---|---|---|---|
| Анализ модуля | ~750K | ~175K | **77%** |
| Исследование метаданных | ~500K | ~130K | **74%** |
| Иерархия вызовов | ~400K | ~90K | **78%** |
| Валидация запроса | ~160K+ | ~20K | **87%** |
| Текстовый поиск | ~44K | ~45K | 0% (паритет) |

---

## Почему Grep проигрывает

1. **False positives** — текстовый поиск находит комментарии, строковые литералы, похожие имена
2. **Iterative deepening** — нужно много уточняющих запросов
3. **Неполный контекст** — прочитал 200 строк, нужное на 350, ещё один запрос

MCP возвращает полный ответ за один вызов.
