# SOUL.md — Личность Hermes Agent

**Источник:** [Статья Влада Смирнова](https://telegra.ph/SOULmd-v-Hermes-Agent-kak-50-strok-zadayut-harakter-agenta-06-13) (адаптация)

## Что такое SOUL.md

SOUL.md — markdown-файл, который Hermes загружает как **основную идентичность агента**. Если файл есть, его содержимое заменяет встроенную идентичность по умолчанию и ставится в первый слот системного запроса — влияет на все последующие слои (инструменты, навыки, память, проектные правила, платформу).

Лежит в HERMES_HOME, **не** в проекте:

| Профиль | Путь |
|---|---|
| default | `~/.hermes/SOUL.md` |
| researcher | `~/.hermes/profiles/researcher/SOUL.md` |
| ops | `~/.hermes/profiles/ops/SOUL.md` |

> Важно: изменения видны в **новой** сессии. Уже идущая продолжает со старым системным запросом.

## Место в системном запросе

Сборка системного запроса по слоям:

1. **Стабильный слой** — SOUL.md, поведение с инструментами, индекс навыков, подсказки окружения/платформы
2. **Контекстный слой** — system_message, проектные контекстные файлы (AGENTS.md / .hermes.md / CLAUDE.md)
3. **Изменяемый слой** — снимок памяти, профиль пользователя, внешняя память, время, сессия, модель

SOUL.md задаёт интерпретационную рамку: если написано «ты строгий ревьюер кода» — агент иначе читает AGENTS.md, навыки и результаты инструментов.

## Что писать в SOUL.md

**Хорошо помещать:**
- идентичность: роль и отношение к пользователю
- голос: тон, длина ответов, степень прямоты
- ценности: что оптимизировать и чего избегать
- поведенческие границы: когда действовать самому, когда спрашивать
- постоянные принципы работы (верны во всех проектах)

**Не класть сюда:**
- проектные инструкции и команды → AGENTS.md
- стиль кода и структура репозитория → .cursorrules
- многошаговые процедуры → навыки (skills)
- факты о пользователе и окружении → память
- провайдеры, модели, инструменты → config.yaml

## Структура

```
# Soul
[1-2 предложения: кто этот агент]
## Голос
[3-5 строк: как общается, тон, длина]
## Работа
[3-5 строк: автономность, принятие решений]
## Ограничения
[3-5 строк: что никогда не делает, когда спрашивает]
```

## Шаблоны ролей

В статье приведены 5 шаблонов: **Стратегический сооснователь**, **Исследователь-аналитик**, **Автономный DevOps**, **Контент-стратег**, **Финансовый аналитик**. Полные тексты — в [оригинале](https://telegra.ph/SOULmd-v-Hermes-Agent-kak-50-strok-zadayut-harakter-agenta-06-13).

## Команды

| Команда | Что делает |
|---|---|
| `hermes prompt-size` | размер системного запроса |
| `/personality NAME` | временный режим (поверх SOUL.md) |
| `/personality` | сброс к базовому SOUL.md |
| `hermes profile create NAME` | новый профиль со своим SOUL.md |
| `hermes profile install URL` | установка дистрибутива профиля |

## Рабочий размер

- **20 строк** — хороший старт
- **50-80 строк** — рабочий диапазон
- **100+ строк** — повод проверить повторы

## Ключевые выводы

- SOUL.md — маленький файл с большим рычагом. Одна строка может изменить весь тон агента
- Не заменяет остальную контекстную инженерию — только рамка поведения
- delegate_task **не наследует** SOUL.md родителя (воркеры используют стандартную идентичность)
- Лучший SOUL.md выращивается итеративно: начать с 20 строк, править по реальному поведению
