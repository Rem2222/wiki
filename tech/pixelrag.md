# PixelRAG

**GitHub:** https://github.com/StarTrail-org/PixelRAG
**Статья:** приложена пользователем в чате (Telegram, 20.06.2026)
**API:** https://api.pixelrag.ai (без API-ключа, предсобранный индекс Wikipedia 8.28M страниц)
**Лицензия:** Apache-2.0
**Звёзды:** ~1.8k (на момент сохранения)

## Суть

Опенсорсный ретривер-фреймворк, который работает с документами как с изображениями, а не текстом. Вместо HTML-to-text пайплайна (теряющего до 40% контента — таблицы, графики, разметку) — рендерит страницу/PDF в скриншоты-тайлы и индексирует их визуально.

## Пайплайн

1. **Рендер** — документ рендерится в тайлы (скриншоты) через Playwright/CDP
2. **Эмбеддинги** — Qwen2-VL-Embedding (дообученный LoRA на скриншотах) → вектор
3. **Индекс** — FAISS
4. **Поиск** — гибридный (визуальная + текстовая схожесть)
5. **Чтение** — VLM-модель (любая, сменная без переиндексации)

## Ключевые особенности

- Замена модели-чтеца не требует переиндексации (в индексе только пиксели)
- Предсобранный индекс Wikipedia — 30M+ скриншотов
- По бенчмаркам авторов: +18.1% к точности против лучшего текстового RAG
- Плагин для Claude Code (/screenshot, визуальный анализ через скриншоты без DOM)
- Работает через pip install pixelrag[serve|index|embed]

## Ограничения

- Требует GPU (Qwen2-VL-2B для эмбеддингов)
- Не является заменой веб-скрейпингу — это про retrieval, а не про извлечение данных
- На CPU не полетит
- Плагин под Claude Code, не под Hermes

## Применимость к текущей инфраструктуре

Для 1С Help Server (RAG по справке), поиска по wiki и коду — избыточен (текстовый RAG через GBrain справляется без GPU). Актуален для задач с критически важной визуальной информацией: дашборды, графики, таблицы в PDF, слайды презентаций.
